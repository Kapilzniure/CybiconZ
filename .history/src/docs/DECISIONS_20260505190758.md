# CybiconZ — Design Decisions Log
## Why things are the way they are. Don't change without good reason.

---

## Hero Section
DECISION: Browser mockup showing LwangBlack project instead of Three.js object
REASON: A rotating abstract shape communicates nothing about CybiconZ.
A real project mockup communicates "these people build real things."
Aura comes from credibility, not decoration.

## No Icon Library
DECISION: All icons are inline SVG
REASON: Keeps bundle small. Full control over stroke weight.
The arrows used are 3-5 lines of SVG — not worth a dependency.

## Two Accent Colors Max
DECISION: Violet-to-pink gradient (primary) + section-specific accents
REASON: More than 2 dominant colors creates visual noise.
The gradient reads as one color family — it's cohesive.
Section accents (amber, cyan, emerald) each appear in one section only.

## Section Alternation
DECISION: Dark → Light → Dark → Light strictly enforced
REASON: Cornor Tech's visual intelligence. Each section feels like
a different room — same house, different mood. Breaks monotony.
Prevents the "all dark" problem that made previous versions feel lifeless.

## CSS 3D on Service Cards
DECISION: perspective(900px) rotateY(-15deg) CSS transform
REASON: Three.js for service cards would be overkill and slow.
CSS perspective creates genuine 3D depth. No JS overhead.

## No Lorem Ipsum Ever
DECISION: Real content or realistic placeholder + // TODO comment
REASON: Lorem ipsum causes AI to stop thinking about the actual content.
It also means design decisions are made without considering real text length.

## DM Mono for Labels
DECISION: All section eyebrows, stats, metadata use DM Mono
REASON: Creates visual hierarchy between "what type of content is this"
and "the content itself." Monospace reads as precise and technical.

## Framer Motion once:true
DECISION: All scroll animations trigger only once
REASON: Animations that repeat every time you scroll back up are annoying
after the first visit. Once is discovery. Twice is noise.

## Testimonials Without Full Names
DECISION: Placeholder names until real testimonials collected
REASON: "Sarah, CEO" looks fake. Nothing looks honest.
Waiting for real ones is the right call.

## Architecture

**Stack:** React 18 + TypeScript, Vite, Tailwind CSS v3, shadcn/ui (Radix UI), Framer Motion, React Router v6, TanStack Query, Three.js / React Three Fiber.

**Page layout pattern:** Every page wraps its content in `<SiteShell>` (`src/components/site/SiteShell.tsx`), which composes `AnnouncementBar → Navbar → PageWrapper → Footer`. `PageWrapper` applies the shared Framer Motion page-transition animation. Route transitions use `AnimatePresence` in `App.tsx`.

**Component layers:**
- `src/components/site/` — layout chrome (Navbar, Footer, SiteShell, etc.)
- `src/components/sections/` — homepage section components (Hero, Services, Portfolio, etc.)
- `src/components/ui/` — shadcn/ui primitives; do not edit these manually; use `npx shadcn@latest add <component>` to add new ones
- `src/pages/` — one file per route, thin; content comes from section components and `src/data/`

**Data layer:** `src/data/projects.ts` and `src/data/services.ts` are the single source of truth for portfolio work and service offerings. Pages import directly from these files — there is no API or CMS yet.

## Design system

The color system lives in `src/index.css` CSS custom properties and is aliased in `tailwind.config.ts`. Use these semantic tokens rather than raw Tailwind colors:

| Token | Usage |
|---|---|
| `bg-brand-base` / `brand-card` / `brand-raised` | dark surface layers |
| `surface-light` (utility class) | off-white sections |
| `text-ink` / `ink-muted` / `ink-dark` | text hierarchy |
| `violet` / `pink` / `cyan` / `amber` / `emerald` | accent palette |
| `.text-gradient` | violet→pink gradient text |
| `.bg-accent-gradient` | same gradient as background |
| `.label-eyebrow` | mono uppercase section labels |
| `.grid-overlay` | subtle grid background pattern |

**Typography:** `font-display` = Bricolage Grotesque (headings), `font-sans` = Plus Jakarta Sans (body), `font-mono` = DM Mono (labels/eyebrows). All loaded via `@fontsource` — no Google Fonts requests.

**Path alias:** `@/` maps to `src/`.

## Adding content

- **New project:** add an entry to `src/data/projects.ts`. The `slug` becomes the `/work/:slug` URL. Fill `situation`, `approach`, and `delivered` fields; both are currently placeholder text marked `// TODO`.
- **New service:** add to `src/data/services.ts`. Services are displayed on `/services` and referenced in other sections.
- **New page:** create `src/pages/MyPage.tsx`, wrap in `<SiteShell>`, add a `<Route>` in `App.tsx`.
